# NodeAgent Self-Healing - Reconciliation Loop

## 0. 문서의 목적
- **목표**: 주기적으로 actual state와 desired state를 비교하여 종료된 컨테이너를 감지하는 기능
- **왜 필요한가**: 컨테이너가 비정상 종료되었을 때 이를 감지하고 재시작 처리를 시작해야 함
- 이 문서는 NodeAgent의 manager.rs에 reconciliation_loop 함수를 추가하기 위해 작성되었습니다.
- 이 기능은 이 문서에 포함된 조건 및 규칙들을 따라야 합니다.

## 1. NodeAgent Manager의 기능
- **gather_container_info_loop**: 컨테이너 상태 수집 (기존)
- **gather_node_info_loop**: 노드 리소스 수집 (기존)
- **reconciliation_loop**: Actual vs Desired 비교 및 복구 처리 (신규)
- **probe_loop**: Liveness probe 실행 (신규)

## 2. Manager의 구현 구조
- **manager.rs**: 모든 백그라운드 태스크 실행 및 관리, 전역 메모리 캐시 `Arc<Mutex<HashMap<String, DesiredState>>>` 보유
- **desired_state.rs**: DesiredState 구조체 정의
- **runtime/podman/container.rs**: 컨테이너 상태 조회 (inspect)

## 3. Reconciliation Loop를 위해 Manager에 구현되어야 하는 것

```
+-------------------+         +---------------------+         +-------------------+
|  Memory Cache     |  read   |   NodeAgent         | inspect |      Podman       |
| HashMap<String,   | <------ | (reconciliation_    | <-----> |  (actual_state)   |
|  DesiredState>    |         |  loop)              |         |                   |
+-------------------+         +---------------------+         +-------------------+
                                        |
                                        | 종료 컨테이너 발견 시
                                        v
                              handle_exited_container()
```

- **인터페이스**: 메모리 캐시에서 조회, Podman API 조회
  - **입력**: 
    1. 메모리 캐시에서 모든 desired state 조회 (`desired_states_cache.lock().await`)
    2. Podman에서 모든 컨테이너 actual state 조회
  - **처리**:
    1. 매 1초마다 실행
    2. Desired state 목록과 Actual state 비교
    3. Desired에 있지만 Actual에 없거나 exited/dead 상태인 컨테이너 발견
    4. 해당 컨테이너에 대해 `handle_exited_container()` 호출
  - **출력**: 종료 컨테이너 감지 시 handle_exited_container 실행

## 4. 지켜야 할 규칙
- **실행 주기**: 1초마다 실행
  ```rust
  pub async fn reconciliation_loop(
      desired_states_cache: Arc<Mutex<HashMap<String, DesiredState>>>
  ) {
      loop {
          // 로직 실행
          tokio::time::sleep(Duration::from_secs(1)).await;
      }
  }
  ```
- **비교 대상**: 
  - Desired: 메모리 캐시 `HashMap<String, DesiredState>`
  - Actual: Podman inspect로 조회한 컨테이너 상태
- **메모리 캐시 조회 방식**:
  ```rust
  // Clone해서 lock 즉시 해제 (동시성 향상)
  let desired_states = {
      let cache = desired_states_cache.lock().await;
      cache.clone()
  };
  // 이제 lock 없이 사용 가능
  for (pod_name, desired_state) in desired_states {
      // 처리
  }
  ```
- **종료 판단 조건**: 
  - `state.Status == "exited"` 
  - 또는 `state.Status == "dead"` 
  - 또는 컨테이너가 존재하지 않음 (inspect 실패)
- **에러 처리**: Podman 조회 실패 시 로그 남기고 다음 루프 계속 진행

## 5. 구현 예시
```rust
async fn reconciliation_loop(
    desired_states_cache: Arc<Mutex<HashMap<String, DesiredState>>>
) {
    loop {
        // 1. 메모리 캐시에서 desired states 조회 (clone으로 lock 즉시 해제)
        let desired_states = {
            let cache = desired_states_cache.lock().await;
            cache.clone()
        };

        // 2. Podman에서 actual states 조회
        let actual_containers = match podman::list_containers().await {
            Ok(containers) => containers,
            Err(e) => {
                eprintln!("Failed to list containers: {:?}", e);
                tokio::time::sleep(Duration::from_secs(1)).await;
                continue;
            }
        };

        // 3. Desired vs Actual 비교
        for (pod_name, desired) in desired_states {
            let actual = actual_containers.iter()
                .find(|c| c.id == desired.container_id);

            match actual {
                None => {
                    // 컨테이너가 존재하지 않음
                    handle_exited_container(&desired, 0).await;
                }
                Some(container) if container.state == "exited" || container.state == "dead" => {
                    // 컨테이너가 종료됨
                    let exit_code = container.exit_code.unwrap_or(1);
                    handle_exited_container(&desired, exit_code).await;
                }
                _ => {
                    // 정상 실행 중
                }
            }
        }

        tokio::time::sleep(Duration::from_secs(1)).await;
    }
}
```

## 6. DesiredState 필드별 참조 위치

| 필드 | 이 문서(Reconciliation Loop)에서 사용 | 다른 문서에서 사용 |
|------|--------------------------------------|------------------|
| `pod_name` | ✅ HashMap의 key로 사용 | 1번 문서: 저장 시 key |
| `container_id` | ✅ Podman inspect 호출 시 사용 | 3번: probe_loop<br>4번: handle_exited_container |
| `restart_policy` | ❌ 사용 안함 | 4번: handle_exited_container에서 재시작 여부 판단 |
| `probe_config` | ❌ 사용 안함 | 3번: probe_loop에서 liveness probe 설정 참조 |
| `created_at` | ⚠️ 로깅/디버깅용 (선택) | 모든 곳에서 로깅 시 사용 가능 |

### Reconciliation Loop에서 실제 사용하는 필드:
1. **pod_name**: HashMap 순회 시 key로 사용
2. **container_id**: `podman::inspect(&desired.container_id)` 호출하여 actual state 조회

→ `restart_policy`, `probe_config`는 이 루프에서 직접 사용하지 않고, 발견한 종료 컨테이너를 `handle_exited_container()`에 전달할 때 DesiredState 전체를 넘겨서 4번 문서에서 사용합니다.
